# CHINT ETM MDM

CHINT ETM MDM помогает подготовить файл `upload_goods` для ETM: заполняет
данные по артикулам из локальной базы, показывает незаполненные поля, обновляет
базу из прайса и ETIM-файлов, а также запоминает ручные дозаполнения.

Программа рассчитана на обычную работу без Python, терминала и прав
администратора. Пользователю достаточно распаковать portable-архив и запустить
`CHINT ETM MDM.exe`.

## Быстрый запуск

1. Распакуйте архив `CHINT_ETM_MDM_portable.zip` в удобную папку.
   Например, в `Документы` или на рабочий стол.
2. Откройте папку `CHINT ETM MDM`.
3. Запустите `CHINT ETM MDM.exe`.
4. На вкладке `База` выберите рабочую папку.
5. Выберите SQLite-базу `chint_mdm.sqlite`.
6. Перейдите на вкладку `Заполнение upload_goods` и выберите шаблон
   `upload_goods.xlsx`.
7. Нажмите `Заполнить из базы`.

Если Windows покажет предупреждение о неизвестном издателе, нажмите
`Подробнее` -> `Выполнить в любом случае`. Это обычное предупреждение для
внутренних portable-программ без цифровой подписи.

## Рабочая папка

Рабочая папка - это место, где программа хранит базу, отчеты, скачанные прайсы
и пользовательские правила. Лучше выбрать отдельную папку, например:

```text
Документы\CHINT ETM MDM Work
```

Внутри нее программа использует такие подпапки:

```text
database\          база данных
output\            заполненные шаблоны и отчеты по заполнению
reports\           отчеты импортов
downloads\price\   скачанные прайс-листы
rules\             правила маппинга и ручные дозаполнения
```

## Заполнение upload_goods

1. На вкладке `База` проверьте рабочую папку и путь к базе.
2. На вкладке `Заполнение upload_goods` выберите исходный шаблон.
3. Проверьте папку результата. Обычно это `output` внутри рабочей папки.
4. Нажмите `Заполнить из базы`.
5. Откройте созданный файл `_filled.xlsx`.

Что важно:

- заполненные значения берутся из базы;
- выпадающие списки из исходного шаблона сохраняются;
- обязательные незаполненные ячейки подсвечиваются красным;
- рядом создается отчет, где видно, что заполнено и что требует ручной работы.

## Правила маппинга

Некоторые поля в шаблонах ETM называются не так, как характеристики в базе.
Для этого есть правила маппинга.

Если после заполнения остались поля `Конфиг:*` или похожие незаполненные
характеристики:

1. Нажмите `Проанализировать маппинг`.
2. Перейдите на вкладку `Правила маппинга`.
3. Посмотрите предложенные источники и примеры значений.
4. Нажмите `Принять` только у тех строк, где источник точно подходит по смыслу.
5. Нажмите `Отклонить` у неподходящих вариантов.
6. Снова запустите `Заполнить из базы`.

Принятые и отклоненные правила сохраняются в рабочей папке и используются в
следующих заполнениях.

## Ручные дозаполнения

Если файл `_filled.xlsx` дозаполнили вручную, эти значения можно сохранить,
чтобы программа подставляла их сама в следующий раз.

1. Заполните шаблон через программу.
2. Дозаполните красные поля вручную в полученном `_filled.xlsx`.
3. В блоке `Ручные дозаполнения` выберите этот дозаполненный файл.
4. Нажмите `Импортировать ручные значения`.
5. Проверьте отчет импорта.

Программа сохраняет только те значения, которые отличаются от данных в базе.
При следующем заполнении они применяются поверх базы.

Файл ручных значений хранится здесь:

```text
<рабочая папка>\rules\manual_values.json
```

## Обновление прайса

На вкладке `Обновление базы` можно обновить базу из прайс-листа CHINT.

Обычный сценарий:

1. Нажмите `Найти свежий прайс`.
2. Нажмите `Скачать и импортировать`.
3. Дождитесь сообщения об окончании импорта.
4. Проверьте статистику в журнале.

Если прайс уже скачан:

1. Выберите файл `Price-list-CHINT_*.xlsx`.
2. Нажмите `Импортировать выбранный XLSX`.

Перед изменением базы программа создает бэкап рядом с файлом базы.

## Импорт ETIM

ETIM-файл нужен для загрузки характеристик и габаритов.

1. На вкладке `Обновление базы` выберите ETIM-файл `.xlsx` или `.xlsm`.
2. Нажмите `Импортировать ETIM`.
3. Дождитесь завершения импорта.
4. Откройте созданный отчет кнопкой `Открыть отчет ETIM`.

Программа ищет артикулы из базы во всех листах ETIM-файла, записывает найденные
характеристики и дозаполняет пустые габариты.

Уже заполненные габариты не перезаписываются автоматически. Если в базе уже
есть значение, а из ETIM пришло другое, строка попадает в отчет `Конфликты`.

## Разбор конфликтов ETIM

Откройте отчет ETIM и перейдите на лист `Конфликты`.

В первой колонке `Решение` выберите один из вариантов:

- `Оставить базу` - ничего не менять;
- `Принять ETIM` - записать значение из ETIM в базу.

После заполнения решений:

1. Сохраните Excel-файл отчета.
2. Вернитесь в программу.
3. Нажмите `Применить решения ETIM`.

Программа применит только строки, где явно выбрано `Принять ETIM`. Перед этим
она создаст новый бэкап базы.

## Откат базы из бэкапа

Если после импорта что-то пошло не так:

1. На вкладке `База` выберите текущую базу.
2. В поле бэкапа выберите нужный файл `.bak_*`.
3. Нажмите `Откатить базу из бэкапа`.

Перед откатом программа сохраняет текущую базу в отдельный файл
`.before_restore_*`, чтобы можно было вернуться назад.

## Обновление программы

Если вам передали новую portable-версию:

1. Закройте программу.
2. Распакуйте новый архив в отдельную папку.
3. Запустите новый `CHINT ETM MDM.exe`.
4. Выберите прежнюю рабочую папку и прежнюю базу.

Рабочую папку удалять не нужно: в ней лежат база, правила, отчеты и ручные
дозаполнения.

## Если программа не запускается

Проверьте по порядку:

- архив полностью распакован, программа не запускается прямо из zip;
- путь к папке не слишком длинный;
- папка доступна на запись;
- антивирус не поместил `.exe` или соседние файлы в карантин;
- база не открыта другой программой в режиме блокировки.

Если ошибка повторяется, передайте текст ошибки и опишите, на каком действии
она появилась.

## Сборка portable-версии

Этот раздел нужен только тому, кто собирает программу для передачи другим
пользователям.

Для сборки нужен установленный Python 3.10 или новее. Права администратора не
требуются: скрипт создает локальную `.venv` внутри папки проекта.

На Windows запустите:

```bat
build_windows.bat
```

Скрипт:

- перейдет в папку проекта;
- создаст `.venv`, если ее еще нет;
- установит зависимости и PyInstaller;
- соберет приложение в `dist\CHINT ETM MDM`;
- создаст архив `dist\CHINT_ETM_MDM_portable.zip`.

Пользователям нужно передавать именно архив `CHINT_ETM_MDM_portable.zip`.
Python, `.venv`, исходники проекта и права администратора им не нужны.
